What is MUCK?

MUCK is software for making sense out of massive, messy collections of documents. This toolkit can be used to take directories of unwieldy amounts of texts, extract useful information (both metadata and linguistic features like parts-of-speech, named entities, and parsed dependencies), and construct a comprehensible database with all extracted information readibly available. You can also use this toolkit to build classifiers from random subsets of this database (as well as classify the rest of the documents in the database) relatively easily...just not quite yet.

History

MUCK was initially written to handle large collections of news content and government data, such as newspaper archives, Congressional press releases/floor speeches, and legislation. The toolkit was designed to answer many different questions that required natural language understanding of these texts, such as which Congressmen get the most news attention and under what conditions.

Authors

MUCK was created by Nick Nugent, Rebecca Weiss, and Sean Westwood.

Version

Currently not even in alpha with no immediate plans for release. Use at your own risk!

Tech

MUCK uses a lot of open-source libraries:

  • Apache Software Foundation - Lucene for text indexing
  • Stanford NLP group - JavaNLP and Classifier for natural language processing and machine learning capability.
  • 10Gen - documents are stored in MongoDB

MUCK was designed for use on AWS platforms, and can scale up by distributing across EC2 instances. This means that you can add and remove nodes depending on your task.

MUCK is also intended to complement Amazon Mechanical Turk, primarily for creating training data needed in classification tasks (topics, sentiment, etc).

Installation

You can git clone git@github.com:rjweiss/MUCK-toolkit.git. But you probably won't want to, since you will need to download and maintain your own dependencies and build from source on your own system. If you run into problems, you are unfortunately on your own. We break this toolkit on a regular basis with our own projects, so there's no guarantee that we will be able to solve problems encountered on other people's systems.
